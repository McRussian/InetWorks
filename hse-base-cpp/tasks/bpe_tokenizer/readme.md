# BPE-токенизатор

Все современные языковые модели (ChatGPT, LLaMA, DeepSeek) работают не с отдельными символами
и не с целыми словами, а с *токенами* — частями слов. Один из основных алгоритмов построения
словаря токенов — **BPE** (Byte Pair Encoding).

Идея проста: начинаем с отдельных символов и итеративно объединяем наиболее частую пару
соседних токенов в один новый токен. В результате частые слова и подслова становятся
одним токеном, а редкие слова разбиваются на подслова.

Реализуйте класс `BPETokenizer`, который обучается на текстовом корпусе и умеет
разбивать слова на подслова.

## Интерфейс

```cpp
class BPETokenizer {
public:
    void Train(std::string_view text, int num_merges);
    std::vector<std::string> Encode(std::string_view word) const;
    const std::vector<std::pair<std::string, std::string>>& GetMergeRules() const;
};
```

- `Train(text, num_merges)` — обучает токенизатор на корпусе `text` (слова разделены пробелами).
  Выполняет не более `num_merges` слияний. Повторный вызов сбрасывает предыдущее обучение.
- `Encode(word)` — разбивает одно слово (без пробелов) на токены, используя обученные правила.
- `GetMergeRules()` — возвращает упорядоченный список правил слияния.

## Алгоритм обучения

1. Разбить корпус на слова по пробелам. Посчитать частоту каждого уникального слова.
2. Представить каждое слово как последовательность односимвольных токенов.
3. Повторить `num_merges` раз:
   1. Для каждой пары соседних токенов `(left, right)` подсчитать суммарную частоту:
      для каждого слова — количество вхождений пары в текущую последовательность токенов,
      умноженное на частоту слова в корпусе.
   2. Выбрать пару с наибольшей суммарной частотой.
      При равенстве — лексикографически наименьшую (сначала по `left`, затем по `right`).
   3. Если пар нет (все слова состоят из одного токена), остановиться.
   4. Заменить все вхождения `(left, right)` на `left + right` во всех словах.
   5. Записать правило слияния.

## Алгоритм кодирования

1. Разбить слово на односимвольные токены.
2. Для каждого правила слияния `(left, right)` (в порядке обучения):
   - Пройти по последовательности токенов слева направо.
   - Если текущий токен равен `left`, а следующий — `right`, заменить их на `left + right`
     и продвинуться на 2 позиции.
   - Иначе — продвинуться на 1.
