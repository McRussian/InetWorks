# Поисковый движок

Реализуйте класс `SearchIndex` — простой полнотекстовый поисковый движок.

## Интерфейс

```cpp
class SearchIndex {
public:
    // Добавляет документ в поисковый индекс.
    // Документ (string_view) должен жить не меньше, чем объект SearchIndex.
    void Add(std::string_view document);

    // Возвращает до max_results наиболее релевантных документов
    // по убыванию релевантности.
    std::vector<std::string_view> Query(std::string_view query, size_t max_results) const;
};
```

## Документы и слова

- **Документом** является строка, переданная в `Add`.
- **Словом** считается максимальная по включению непрерывная последовательность символов, удовлетворяющая критерию `std::isalpha()`. Например, в строке `"hello,world 123abc"` три слова: `hello`, `world`, `abc`.
- Алгоритм **не чувствителен к регистру**: `"Cat"`, `"cat"` и `"CAT"` — одно и то же слово.
- Документы, не содержащие ни одного слова, игнорируются (не учитываются в подсчёте общего числа документов).

## Алгоритм TF-IDF

Мерой релевантности документа будем считать сумму [TF-IDF](https://ru.wikipedia.org/wiki/TF-IDF) по **уникальным** словам запроса.
Если запрос содержит повторы (например, `"cat cat dog"`), каждое слово учитывается один раз.

### Формулы

Пусть:
- `N` — общее количество документов, содержащих хотя бы одно слово
- `df(t)` — количество документов, в которых встречается слово `t`
- `tf(t, d)` — частота слова `t` в документе `d`

```
TF(t, d) = (число вхождений t в d) / (общее число слов в d)
IDF(t) = log(N / df(t))
```

Релевантность документа `d` запросу из уникальных слов `{q_1, ..., q_k}`:
```
relevance(d) = sum(TF(q_i, d) * IDF(q_i))
```

Для логарифма используйте `std::log` (натуральный логарифм).

### Важные частные случаи

- Если слово встречается **во всех** документах, то `IDF = log(1) = 0` — оно не влияет на ранжирование. Это осмысленно: слова вроде "the" не помогают различать документы.
- Слова запроса, не встречающиеся ни в одном документе, не вносят вклада в релевантность.
- Если у двух документов одинаковая релевантность, они возвращаются в порядке добавления.
- Если релевантных документов меньше, чем `max_results`, возвращаются только релевантные (с релевантностью > 0).

## Производительность

Индекс может содержать много документов. Постарайтесь не пересчитывать всё при каждом вызове `Search`.
Возвращаемые `string_view` должны указывать в строки, переданные в `Add` (не копировать документы).

## Пример

```cpp
SearchIndex index;
index.Add("the cat sat on the mat");
index.Add("a dog chased a cat");
index.Add("birds fly in the sky");

auto results = index.Query("cat", 2);
// results[0] == "a dog chased a cat"     (TF=1/5, IDF=log(3/2))
// results[1] == "the cat sat on the mat"  (TF=1/6, IDF=log(3/2))
```

`"a dog chased a cat"` ранжируется выше, потому что `TF(cat) = 1/5 > 1/6`.
